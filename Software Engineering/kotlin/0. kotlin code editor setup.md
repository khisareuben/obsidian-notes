
## 🛠️ How to Run Kotlin Code Without Android Studio

Since Android Studio is heavy, here’s a lightweight setup you can use with Lapce (or VS Code):

### 1. Install Kotlin Compiler

On Zorin OS (Ubuntu‑based):


```zsh
sudo apt update
sudo apt install snapd
sudo snap install kotlin --classic
```

Now you can compile Kotlin files:


```zsh
kotlinc hello.kt -include-runtime -d hello.jar
java -jar hello.jar
```

### 2. Use Gradle for Projects

Install Gradle:


```zsh
sudo apt install gradle
```

Gradle lets you manage Kotlin projects without Android Studio. You can build apps, run tests, and manage dependencies from the terminal.

### 3. Configure Lapce

- Open your Kotlin project folder in Lapce.
    
- Use the **terminal inside Lapce** to run `kotlinc` or `gradle build`.
    
- Lapce gives you syntax highlighting and editing, while the terminal handles compilation and execution.
    

### 4. Optional: Language Server

For better Kotlin support (autocomplete, linting), you can install the **Kotlin Language Server**:



```zsh
git clone https://github.com/fwcd/kotlin-language-server.git
```

Then configure Lapce to use it. This gives you IDE‑like features without Android Studio.

## ✅ Why This Works for You

- **Lightweight**: No need for Android Studio’s heavy UI.
    
- **Offline friendly**: Kotlin compiler + Gradle run locally, no constant network dependency.
    
- **Flexible**: You can use Lapce or VS Code as your editor, and the terminal for builds.
